## Getting Started

1. If you're new to GitHub, consider going through the [GitHub Flow Guide](https://guides.github.com/introduction/flow/).
2. Fork this repository.
3. Clone your forked repository locally.
4. Set up the development environment following instructions in our README.
5. Make your changes and submit a pull request!

## How to Contribute

### Good First Issues

If you're new to our project, we have labeled some issues as [`good first issues`](link-to-your-repo-issues-labeled-good-first-issue) that are great for beginners. These are typically smaller tasks that help you get familiar with the project and the contribution process.

### Pull Request Process

1. Ensure you've branched off from the most recent version of `main` or the default branch of this repo.
2. Ensure any install or build dependencies are removed before the end of the layer when doing a build.
3. Update the README.md with details of changes when applicable.
4. Increase the version numbers in any examples files and the README.md to the new version that this Pull Request would represent. Adherence to [SemVer](http://semver.org/) is required.
5. Submit your pull request! PRs will be reviewed by maintainers and comments/feedback will be given where necessary.
